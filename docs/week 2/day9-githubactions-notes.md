Github Action is a form of workflow automation, there are several actions that can be done and one of them is ci / cd. ci / cd is ci as continues integration which means the continuation of integrity and cd as continues deployment which means the continuation of publishing. So in general, github actions have several actions, one of which is the ci / cd pipeline which will automate all sources that will be owned until it is deemed to have no conflicts, then it will be combined in the main / master branch.
