In the GitHub workflow, create a branch, make changes, make pull requests, review comments during pull requests, merge requests, and delete branches.

Create a branch, for example you create a branch to try out a certain feature before it is published in the main/master branch.

For example, if you make changes in your local Git or online Git (GitHub), then in your local Git you have to do a `git push` for every change that is in Git, and in GitHub you also have to push changes from your local Git.

You can create a pull request from another branch and also add comments during the pull request process so you can know what changes will be incorporated into the main/master branch.

A merge request is made from another branch to the main branch if there are no related issues in the same or different files. A merge request will only be executed if a feature from another branch has been checked. Then, we will merge the tested feature into the main/master branch.

Delete the branch when you're done, use it for testing features.
