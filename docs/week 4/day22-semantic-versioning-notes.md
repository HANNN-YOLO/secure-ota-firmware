Today I'm going to learn about semantic versioning.

So, semantic versioning is a number used to number versions. For example, if the version is 2.1.4, then 2 is for major, 1 is for minor, and 4 is for patch.
Major versioning is used when a major change occurs that makes the new version no longer compatible with the previous version (breaking changes). Once the major version is reached, the minor version must be reset to 0 and the patch version to 0.
Minor versioning only adds features that are backward compatible.
Patch versioning only fixes bugs that are backward compatible.
